## For Supervisors

### How to Escalate an Issue

1. **Open the App**
   - Login as Supervisor
   - Go to Dashboard

2. **Start Escalation**
   - Tap "üö® Escalate Issue" button at bottom of dashboard
   - OR navigate to Issue Escalation from menu

3. **Select Issue Type**
   - Choose from 8 categories:
     - üë• Manpower Shortage
     - ‚ö†Ô∏è Safety Incident
     - üì¶ Material Delay
     - üíî Material Damage
     - üë∑ Worker Misconduct
     - üîß Equipment Breakdown
     - üìã Site Instruction Change
     - üìù Other Issue

4. **Set Severity**
   - LOW - Minor, can wait
   - MEDIUM - Needs attention soon
   - HIGH - Urgent, affecting work
   - CRITICAL - Emergency, work stopped
   - Check "Immediate Action Required" if urgent

5. **Fill Details**
   - **Title**: Brief summary (required)
   - **Description**: Full details, min 20 characters (required)
   - **Estimated Impact**: How it affects work (optional)
   - **Suggested Solution**: Your recommendation (optional)

6. **Select Project & Target**
   - Choose which project
   - Select who to escalate to:
     - Project Manager (most common)
     - Admin Office
     - Senior Management

7. **Add Photos**
   - Take photos or select from gallery
   - Up to 5 photos
   - Shows issue clearly

8. **Add Notes**
   - Any additional context
   - Previous attempts to resolve
   - Related issues

9. **Submit**
   - Review your escalation
   - Tap "Escalate [SEVERITY] Issue"
   - Confirm submission

10. **Done!**
    - Issue is logged
    - Manager is notified
    - You'll get updates on status

## For Managers

### How to Respond to Escalations

1. **Receive Notification**
   - Push notification on mobile
   - Email for critical issues
   - See issue type and severity

2. **Review Details**
   - Open escalation
   - Read full description
   - View photos
   - Check supervisor's suggestion

3. **Take Action**
   - Acknowledge receipt
   - Update status to IN_PROGRESS
   - Add your notes
   - Coordinate resolution

4. **Close Issue**
   - Mark as RESOLVED when fixed
   - Add resolution notes
   - Or mark as DISMISSED if not valid

### üì¶ Material Delay

### üíî Material Damage

## Severity Levels Guide

### üü¢ LOW
- Minor inconvenience
- Can wait 2-3 days
- No immediate impact
- Example: "Need extra safety cones"

### üü° MEDIUM
- Needs attention within 24 hours
- Some impact on work
- Can work around it temporarily
- Example: "Tool needs maintenance"

### üü† HIGH
- Urgent, needs attention within 8 hours
- Significantly affecting work
- Limited workarounds
- Example: "Material shortage affecting schedule"

### üî¥ CRITICAL
- Emergency, needs immediate attention
- Work stopped or safety risk
- No workarounds possible
- Example: "Equipment breakdown, work halted"

## Common Scenarios

### Scenario 1: Material Delay
**Situation**: Cement delivery delayed by 2 days
**Action**:
1. Escalate as "Material Delay" - HIGH severity
2. Add supplier name and expected date
3. Suggest alternative supplier
4. Mark "Immediate Action Required"
5. Manager arranges backup supplier

### Scenario 2: Equipment Breakdown
**Situation**: Excavator hydraulic failure
**Action**:
1. Escalate as "Equipment Breakdown" - CRITICAL
2. Add photos of equipment
3. Note work is stopped
4. Suggest rental equipment
5. Manager arranges immediate repair/rental

### Scenario 3: Site Instruction Change
**Situation**: Client wants design modification
**Action**:
1. Escalate as "Site Instruction Change" - MEDIUM
2. Describe the change requested
3. Estimate impact on timeline/cost
4. Suggest implementation approach
5. Manager coordinates with client and team

## Troubleshooting

### Can't Submit Escalation
- Check internet connection
- Ensure all required fields filled
- Description must be 20+ characters
- Project must be selected

### Photos Won't Upload
- Check camera permissions
- Ensure photos are under 5MB each
- Maximum 5 photos allowed
- Try taking new photo instead

### No Response from Manager
- Check escalation status
- Wait 2-4 hours for non-critical
- Call manager for critical issues
- Re-escalate if needed
